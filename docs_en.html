C) Installation

‚Üë top

Via Installation Wizard (Recommended):

  1. Upload all project files to your web server root directory (public_html or www)
  2. Open your website URL in a web browser (e.g., https://yourdomain.com)
  3. The system will automatically redirect you to /install
  4. Follow the installation wizard steps:
    • Step 1: Requirements Check
    • Step 2: Permissions Check
    • Step 3: Database Setup
    • Step 4: Admin Account Creation
    • Step 5: Final Configuration
  5. Click "Install" to complete the installation
  6. Login to your dashboard with the admin credentials you created

Installation Wizard Benefits:

  • Automatic server requirements validation
  • Automatic database creation and migration
  • Sample data seeding for testing
  • User-friendly step-by-step guidance
  • Reduced chance of configuration errors

Or Manual Installation:

  1. Upload all project files to your web server root directory
  2. Create a new MySQL or PostgreSQL database on your server
  3. Import the database file located at database/wms_pro.sql
  4. Navigate to your project root directory via SSH or terminal
  5. Copy the environment file: cp .env.example .env
  6. Edit the .env file and configure your database credentials
  7. Run the following commands:
    composer install --optimize-autoloader --no-dev
    npm install
    npm run build
    php artisan key:generate
    php artisan migrate --seed
    php artisan storage:link
    php artisan optimize
  8. Set proper folder permissions (755 for directories, 644 for files)
  9. Configure your web server to point to the public directory

Default Admin Login Information:

Email: superadmin@wms.com

Password: password

Post-Installation Security Steps:

  • Delete or rename the /install directory after successful installation
  • Change default administrator password immediately
  • Set APP_DEBUG=false in production environment
  • Configure HTTPS/SSL certificate for secure connections
  • Secure your .env file
  • Enable firewall and configure proper security rules
  • Regular backup database and files

D) Initial Configuration

‚Üë top

After successful installation, configure your WMS Pro system with these essential settings:

1) General System Settings

Navigate to Settings > General Settings from the main menu

2) Create Your First Warehouse

  1. Go to Master Data > Warehouses
  2. Click the "Add New Warehouse" button
  3. Fill in warehouse details and click Save
  4. Set the warehouse status to Active

3) Configure Product Categories

  1. Navigate to Master Data > Categories
  2. Create main categories for organizing your products
  3. Add subcategories as needed

4) Set Up Units of Measure

  1. Go to Master Data > Units of Measure
  2. Add standard units: Pieces, Box, Carton, Kg, Liter, Meter, etc.
  3. Configure conversion factors if using multiple units

N) API Documentation

‚Üë top

WMS Pro provides RESTful API for integration with external systems using Laravel Sanctum for token-based authentication.

Authentication:

POST /api/v1/auth/login

Authorization: Bearer {your_token}

Available Endpoints:

  • Products: GET, POST, PUT, DELETE
  • Stock: GET current stock, adjustments
  • Purchase Orders: CRUD operations
  • Sales Orders: CRUD operations
  • Goods Receipt: Create and confirm
  • Goods Issue: Create and confirm

Rate Limiting: 60 requests per minute (default), 120 for authenticated users

O) Troubleshooting

‚Üë top

1. "500 Internal Server Error"

  • Check file permissions: chmod -R 755 storage
  • Clear cache: php artisan optimize:clear
  • Check .env configuration

2. "Database connection error"

  • Verify database credentials in .env
  • Ensure database server is running
  • Check if database exists

3. "CSS/JS not loading"

  • Run: npm run build
  • Clear browser cache
  • Verify APP_URL in .env
